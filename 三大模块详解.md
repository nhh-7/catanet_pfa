# CAPANet 三大模块详解

整体上，三者不是孤立堆叠，而是一条递进式主链：

```text
DPR 动态语义原型路由
→ PFSA 渐进聚焦稀疏注意力
→ LMR 低到高多级协同重建
```

其中：

1. `DPR` 回答“哪些 token 应该被组织到一起”。
2. `PFSA` 回答“这些候选 token 之间哪些连接值得保留”。
3. `LMR` 回答“低频结构、中频过渡、高频细节如何递进重建”。

---

## 1. 整体 Block 流程

一个 `CAPANet Block` 可以按照如下流程实现：

```text
Input x
→ LayerNorm
→ DPR 内容路由
→ 全局中心交互分支
→ LMR + PFSA 多级聚焦重建
→ Projection / Conv1x1
→ Residual
→ FFN / ConvFFN
→ Residual output
```

输入特征通常先从图像特征形式：

```text
B x C x H x W
```

展平为 token 序列：

```text
B x N x C
```

其中：

```text
N = H * W
```

为了让注意力聚焦信息在网络深层逐步稳定，建议在相邻 `CAPANet Block` 之间传递三级注意力状态：

```python
attn_state = {
    "lf": low_frequency_attention_prior,
    "mf": mid_frequency_attention_prior,
    "hf": high_frequency_attention_prior,
}
```

这样，每个 Block 的注意力不是从零开始计算，而是在上一层已经形成的聚焦先验基础上继续修正和收敛。

## 2. DPR：动态语义原型路由

### 2.1 模块目标

`DPR` 的核心目标不是做普通空间分块，而是根据输入内容动态组织 token。传统窗口注意力或规则分块通常按照空间位置划分 token，但在超分辨率任务中，真正有参考价值的重复纹理、线条、边缘和结构往往分散在不同空间位置。

因此，`DPR` 应该把空间上分散但语义或纹理相似的 token 重新组织到相同或相邻候选集合中，为后续注意力计算提供更干净的候选范围。

它解决的问题可以概括为：

```text
从“空间邻近组织”转向“内容相似组织”
```

### 2.2 输入与输出

`DPR` 的输入是归一化后的 token 序列：

```text
X ∈ R^{B x N x C}
```

其中：

1. `B` 表示 batch size。
2. `N` 表示 token 数量。
3. `C` 表示通道维度。

`DPR` 推荐输出四类信息：

```text
idx_last
sorted_belong_idx
x_scores
prototypes
```

含义如下：

```text
idx_last           # 用于恢复原始 token 顺序
sorted_belong_idx # 排序后的语义原型标签，用于构造 cluster-aware mask
x_scores          # token 到原型的归属置信度，用于 PFSA 动态焦点率
prototypes        # 动态语义原型，用于后续全局中心交互
```

### 2.3 动态语义原型生成

第一步是根据当前输入特征生成动态语义原型：

```text
P ∈ R^{B x M x C}
```

其中 `M` 是原型数量。每个原型可以理解为当前图像内容中的一个语义或纹理中心，例如建筑窗格、漫画线条、边缘区域、平坦背景、重复纹理等。

这里的关键是“动态”。原型不应只是训练后固定不变的参数，而应随输入图像内容变化。本文推荐采用：

```text
Soft Assignment 内容聚合
+
Prototype Query 稳定槽位 / 轻量 refine
+
Prototype-to-Token 反向路由
```

这种组合方式的优点是：

1. `Soft Assignment` 保证 prototype 主要由当前图像内容生成，而不是依赖固定参数。
2. `Prototype Query` 为不同 prototype 提供稳定的槽位身份，避免多个中心塌缩到同一类纹理。
3. 最终再用生成后的 `prototypes` 反查每个 token 的归属，使 `belong_idx` 和 `x_scores` 与动态中心严格一致。

推荐实现路径如下：

```text
X
→ Linear / MLP 得到 token embedding E
→ MLP(E) 预测 token-to-prototype 初始分配 logits
→ Soft Assignment 聚合得到内容原型 P_content
→ 引入 learnable prototype query 作为槽位锚点 Q_proto
→ 用 Q_proto 或 P_content + Q_proto 对 X 做轻量 cross-attention refine
→ LayerNorm / L2Norm 稳定原型特征
→ 得到最终 prototypes P
→ 用 P 反向计算 token 归属 belong_idx 和 x_scores
```

更具体地，先由 token 自己预测属于不同 prototype 的软分配：

```text
E = f_embed(X)
L = f_assign(E)
S_0 = Softmax(L, dim=M)
```

其中：

```text
E: token embedding，形状 B x N x d
L: 初始分配 logits，形状 B x N x M
S_0: token 到 prototype 的初始软分配，形状 B x N x M
```

为了得到第 `m` 个内容原型，需要对该 prototype 下所有 token 的贡献做归一化：

```text
W_{i,m} = S_0[i,m] / (sum_i S_0[i,m] + eps)
P_content[m] = sum_i W_{i,m} · X_i
```

矩阵形式可以写为：

```text
P_content = Norm(W^T · X)
```

这样，每个 prototype 都是当前图像 token 的加权聚合结果。例如，边缘 token 对某个 prototype 的分配权重更高时，该 prototype 就会变成边缘中心；重复纹理 token 权重更高时，它就会变成纹理中心。

然后引入 `M` 个可学习 prototype query：

```text
Q_proto ∈ R^{M x d}
```

它们不直接充当最终中心，而是提供稳定的原型槽位。可以采用轻量 cross-attention 对内容原型进行 refine：

```text
Q = Linear_q(P_content) + Q_proto
K = Linear_k(X)
V = Linear_v(X)
A_proto = Softmax(QK^T / sqrt(d))
P_refine = A_proto · V
P = Norm(P_content + gamma · P_refine)
```

其中 `gamma` 可以是固定系数，也可以是可学习门控。这样得到的 `P` 同时具备两类性质：

1. 内容上来自当前图像 token 的动态聚合。
2. 结构上受到 prototype query 约束，具有稳定的多中心分工。

因此，`DPR` 中推荐的原型生成不是：

```text
固定 learnable prototypes → 直接匹配 token
```

而是：

```text
当前图像 token → soft assignment 聚合内容中心 → prototype query refine → 动态 prototypes
```

### 2.4 Token 与原型关联建模

第二步是用最终动态原型 `P` 反向计算 token 与原型之间的关联分数：

```text
S = Softmax(phi(X) · psi(P)^T)
```

其中：

```text
phi(X): token 映射后的查询特征，形状 B x N x d
psi(P): prototype 映射后的键特征，形状 B x M x d
S: token 到 prototype 的归属概率，形状 B x N x M
```

对每个 token，取关联分数最高的原型作为语义归属：

```text
belong_idx = argmax(S, dim=M)
x_scores = max(S, dim=M)
```

其中：

```text
belong_idx # 每个 token 所属的语义原型编号
x_scores   # 每个 token 对所属原型的归属置信度
```

`x_scores` 很重要。它不只是中间变量，还会被后续 `PFSA` 用于判断一个 subgroup 内部是否纯净，以及是否应该保留更多注意力连接。

### 2.5 Token 排序与候选集合构建

第三步是根据 `belong_idx` 对 token 进行排序，让属于同一原型的 token 在序列中靠近：

```text
sorted_idx = sort_by(belong_idx, x_scores)
X_sorted = gather(X, sorted_idx)
```

排序逻辑可以有两层：

1. 先按 `belong_idx` 排序，使同一语义原型的 token 聚集。
2. 再按 `x_scores` 排序，使高置信 token 更靠近该组中心。

同时需要保存恢复索引：

```text
idx_last = inverse_index(sorted_idx)
```

后续 `LMR + PFSA` 在排序后的 token 序列上完成交互后，需要通过 `idx_last` 将结果恢复到原始空间顺序。

### 2.6 DPR 的工程注意事项

`DPR` 实现时需要重点注意以下细节：

1. 排序必须在 batch 内独立完成，不能跨 batch 混合。
2. `idx_last` 必须严格对应 `sorted_idx` 的逆操作，否则输出会出现空间错位。
3. `sorted_belong_idx` 要与 `X_sorted` 完全对齐，后续 mask 构造依赖它。
4. `x_scores` 要跟随 token 一起排序，不能仍保留原始顺序。
5. 若后续使用 `2x` 或 `4x` token pooling，需要处理 token 数不能整除的问题。
6. `prototypes` 应显式输出，便于全局中心交互分支复用。

因此，`DPR` 最终不是只输出排序后的特征，而是要输出一组路由元信息：

```text
X_sorted, idx_last, sorted_belong_idx, x_scores, prototypes
```

其中 `X_sorted` 可以在模块内部返回，也可以由后续模块根据 `sorted_idx` 自行 gather。

## 3. PFSA：渐进聚焦稀疏注意力

### 3.1 模块目标

`PFSA` 的作用是在 `DPR` 给出的候选集合中进一步筛选关键连接。

普通注意力会让候选集合内所有 token 两两交互：

```text
N tokens → N x N dense attention
```

但轻量级超分辨率模型中，很多连接价值很低，甚至会引入边界噪声、不相关纹理和错误结构。因此，`PFSA` 应该只保留最重要的 TopK 连接，将计算资源集中到高价值交互上。

它解决的问题可以概括为：

```text
从“候选 token 全连接”转向“关键连接稀疏交互”
```

### 3.2 输入与输出

`PFSA` 的输入包括：

```text
X_level             # 当前层级特征
sorted_belong_idx   # DPR 输出的语义标签
x_scores            # DPR 输出的归属置信度
prev_attn_state     # 上一 Block 或上一阶段的注意力先验
k_global, v_global  # 可选的全局中心交互信息
```

输出包括：

```text
Y_level             # 当前层级注意力输出
current_attn_state  # 当前层级注意力状态，用于传递到下一 Block
```

由于 `LMR` 中存在低频、中频、高频三个尺度，`PFSA` 不应只维护一个注意力状态，而应维护三级状态：

```python
prev_attn_state = {
    "lf": prev_attn_lf,
    "mf": prev_attn_mf,
    "hf": prev_attn_hf,
}
```

### 3.3 当前内容相似性计算

对当前层级特征生成 `Q, K, V`：

```text
Q = XWq
K = XWk
V = XWv
```

计算当前内容相关性：

```text
A_cur = Softmax(QK^T / sqrt(d))
```

这里的 `A_cur` 只表示当前层根据内容相似性计算出的注意力分布，不应直接作为最终注意力。

### 3.4 注意力先验融合

`PFSA` 的“渐进聚焦”体现在：当前层注意力不从零开始，而是继承上一层已经形成的聚焦先验。

融合方式可以写为：

```text
A_hat = beta_{l,s} * A_prev + (1 - beta_{l,s}) * A_cur
```

其中：

```text
A_prev # 上一 Block 或上一阶段的注意力先验
A_cur  # 当前内容相似性注意力
beta_{l,s} # 第 l 个 Block、第 s 个尺度分支的注意力继承权重
A_hat  # 融合后的聚焦分布
```

这里不使用一个全局固定 `beta`。采用：

```text
depth-aware initialization + learnable beta
```

即先按网络深度和尺度分支设置合理初值，再让训练过程微调：

```text
beta_{l,s} = Sigmoid(beta_param_{l,s})
```

其中：

```text
l 表示第几个 CAPANet Block
s 表示尺度分支，s ∈ {lf, mf, hf}
```

推荐关系为：

```text
beta_lf > beta_mf > beta_hf
```

原因是：

1. 低频结构更稳定，可以更多继承上一层注意力先验。
2. 中频边界既需要继承结构，也需要根据当前内容修正。
3. 高频细节变化最大，过强继承容易锁定错误纹理或放大伪影。

如果一共有 `L` 个 `CAPANet Block`，可以按深度线性初始化：

```text
t = l / (L - 1)

beta_lf = 0.35 + 0.40 * t
beta_mf = 0.25 + 0.40 * t
beta_hf = 0.10 + 0.40 * t
```

对应推荐范围为：

```text
低频 lf: 0.35 → 0.75
中频 mf: 0.25 → 0.65
高频 hf: 0.10 → 0.50
```

如果 `L = 1`，则可以直接使用浅层初值：

```text
beta_lf = 0.35
beta_mf = 0.25
beta_hf = 0.10
```

工程实现时，可以先把初始化值转换成 logit，再作为可学习参数：

```python
def init_beta_param(beta_init):
    beta_init = torch.tensor(beta_init)
    return nn.Parameter(torch.logit(beta_init))

self.beta_lf = init_beta_param(beta_lf_init)
self.beta_mf = init_beta_param(beta_mf_init)
self.beta_hf = init_beta_param(beta_hf_init)

beta_lf = torch.sigmoid(self.beta_lf)
beta_mf = torch.sigmoid(self.beta_mf)
beta_hf = torch.sigmoid(self.beta_hf)
```

一般来说：

1. 浅层更依赖当前内容，因此 `A_cur` 权重可以更大。
2. 深层需要稳定聚焦，因此 `A_prev` 权重可以逐渐提高。
3. 若上一层状态为空，则直接使用 `A_cur` 初始化。
4. 不建议一开始把 `beta` 设得过高，例如 `0.85` 以上，否则当前内容很难纠正上一层可能形成的错误注意力。

### 3.5 动态焦点率计算

`PFSA` 不使用固定 TopK，而是根据当前 subgroup 的内容质量动态调整保留比例。

推荐定义：

```text
r_l = clip(
    r_base + lambda_p * purity_l - lambda_v * var_l,
    r_min,
    r_max
)
```

其中：

```text
purity_l # 当前 subgroup 中同类标签的占比
var_l    # 当前 subgroup 内 x_scores 的方差
r_base   # 基础保留比例
r_min    # 最小保留比例
r_max    # 最大保留比例
```

推荐默认值：

```text
r_base = 0.5
r_min  = 0.25
r_max  = 0.75
```

该公式的含义是：

1. `purity_l` 越高，说明 subgroup 内 token 语义越一致，可以保留更多连接。
2. `var_l` 越高，说明 token 置信度波动越大，组内可能混杂，应减少连接。
3. `clip` 保证保留比例不会过高或过低，避免训练不稳定。

例如，某个 subgroup 有 `64` 个 token：

```text
r_l = 0.50 → 每个 query token 保留约 32 个 key
r_l = 0.25 → 每个 query token 保留约 16 个 key
r_l = 0.75 → 每个 query token 保留约 48 个 key
```

### 3.6 TopK 稀疏掩码生成

根据融合注意力分布 `A_hat` 和动态焦点率 `r_l` 生成 TopK 掩码：

```text
M_l = TopK(A_hat, r_l)
```

实现时通常按 query 行生成掩码，即每个 query token 选择自己最相关的一部分 key token。

具体过程：

```text
1. 对 A_hat 的每一行取 TopK
2. TopK 位置设为有效连接
3. 非 TopK 位置设为无效连接
4. 对无效连接填充 -inf
5. 重新执行 Softmax
```

公式可以写为：

```text
A_sparse = Softmax(mask(A_hat, M_l))
Y = A_sparse · V
```

### 3.7 Cluster-aware Mask

除了 TopK 稀疏，还可以利用 `sorted_belong_idx` 构造 cluster-aware mask。

其核心思想是：同一语义原型内的 token 更容易互相连接，不同原型之间的 token 连接需要更谨慎。

一种实现方式是：

```text
base_mask[i, j] = 1, if sorted_belong_idx[i] == sorted_belong_idx[j]
base_mask[i, j] = 0, otherwise
```

然后将它与 TopK mask 结合：

```text
final_mask = TopK_mask OR selected_cross_cluster_mask
```

为了避免完全切断跨组信息，保留少量跨 cluster 的高置信连接，特别是对边界、重复纹理和长距离结构有帮助的连接。

### 3.8 全局中心交互接入

全局中心交互分支的作用是：让 token 在局部稀疏交互之外，额外参考当前图像的动态语义原型。也就是说，`PFSA` 不只建模：

```text
token ↔ token
```

还额外建模：

```text
token ↔ prototype center
```

这里使用独立全局中心分支，而不是直接把 `k_global, v_global` 拼接到局部 `K, V` 中。原因是独立分支更容易控制全局先验的强度，也更方便做消融实验。

全局中心分支不参与 PFSA 的 TopK mask、cluster-aware mask 或注意力状态递推，只在 PFSA 输出后通过 eta_s 做残差式融合，因此它是对 PFSA 的弱耦合补充，而不是 PFSA 稀疏注意力本身的一部分。

具体地，先利用 `DPR` 输出的动态原型：

```text
P ∈ R^{B x M x C}
```

生成全局中心分支的键和值：

```text
k_global = Linear_k_global(P)
v_global = Linear_v_global(P)
```

其中：

```text
k_global ∈ R^{B x M x d}
v_global ∈ R^{B x M x d}
```

局部稀疏注意力仍然由 `PFSA` 正常计算：

```text
Y_local_sparse = PFSA(Q_local, K_local, V_local)
```

然后单独计算 token 到全局 prototype center 的注意力：

```text
A_global = Softmax(Q_local · k_global^T / sqrt(d))
Y_global_center = A_global · v_global
```

最后用尺度相关门控系数融合：

```text
Y = Y_local_sparse + eta_s * Y_global_center
```

其中 `s ∈ {lf, mf, hf}` 表示低频、中频、高频分支。推荐设置为：

```text
eta_lf = 0
eta_mf = 0.1 ~ 0.3
eta_hf = 0.2 ~ 0.5
```

也就是：

1. 低频分支不接入全局中心分支。
2. 中频分支少量接入全局中心分支。
3. 高频分支更强接入全局中心分支。

这样设计的原因是：

1. 低频分支本身负责整体轮廓、大范围结构和长距离关系，再接入全局中心容易重复叠加全局信息，导致结构过度平滑。
2. 中频分支负责边界、区域过渡和局部形状，少量全局中心先验有助于保持结构边界一致。
3. 高频分支负责纹理、边缘和细节，更需要参考全图的重复纹理、线条类型和语义中心，以减少局部细节漂移。

工程上，`eta_s` 可以先使用固定超参数跑通实验；如果希望模型自适应控制全局分支强度，可以改成可学习门控：

```text
eta_s = eta_max_s * Sigmoid(eta_param_s)
```

其中：

```text
eta_max_lf = 0
eta_max_mf = 0.3
eta_max_hf = 0.5
```

因此，本课题的全局中心交互方式可以概括为：

```text
local sparse PFSA
+
independent prototype-center attention
+
scale-aware gated fusion
```

### 3.9 PFSA 的工程注意事项

`PFSA` 实现时需要重点注意：

1. `prev_attn_state` 的形状必须与当前层级 token 粒度匹配。
2. 低频、中频、高频三个尺度应分别维护注意力状态，不能混用。
3. 若序列长度因 pooling 改变，需要对 attention prior 做对应降采样或重新初始化。
4. TopK 掩码生成后，softmax 前应使用数值稳定的 mask fill。
5. 对被 mask 的位置建议填充 `-inf` 或足够小的负数。
6. 稀疏注意力如果暂时没有高效后端，可以先用 dense attention + mask 跑通逻辑，再优化计算。
7. `x_scores` 和 `sorted_belong_idx` 必须与当前 token 顺序完全一致。

## 4. LMR：低到高多级协同重建

### 4.1 模块目标

`LMR` 的核心思想是：超分辨率重建不是直接锐化高频细节，而是应先恢复整体结构，再恢复边界过渡，最后补充高频纹理。

如果所有 attention head 都在同一粒度上竞争式建模，模型容易出现：

1. 主体结构不连续。
2. 边缘位置错位。
3. 高频纹理伪影。
4. 重复模式不稳定。

因此，`LMR` 将路由后的特征显式拆分为低频、中频和高频三个层级，并建立低到高的递进重建链路。

它解决的问题可以概括为：

```text
从“单尺度混合重建”转向“低频到高频递进重建”
```

### 4.2 三路分支拆分

`LMR` 接收 `DPR` 排序后的 token 序列，并按通道或 head 拆分为三路：

```text
X_hf # 高频分支
X_mf # 中频分支
X_lf # 低频分支
```

推荐 head 分配：

```text
高频 X_hf：2 个 head，保持原始 token 粒度
中频 X_mf：1 个 head，执行 2x token pooling
低频 X_lf：1 个 head，执行 4x token pooling
```

对应形状可以理解为：

```text
X_hf: B x N     x C_hf
X_mf: B x N/2   x C_mf
X_lf: B x N/4   x C_lf
```

其中：

1. 高频分支负责边缘、细线条、纹理和重复细节。
2. 中频分支负责结构边界、区域过渡和局部形状。
3. 低频分支负责整体轮廓、大范围结构和长距离关系。

### 4.3 Segment Pooling

中频和低频分支需要对 token 做 pooling。这里不建议直接按原始空间位置池化，而应在 `DPR` 排序后的 subgroup 内做 `segment pooling`。

原因是：`DPR` 已经把语义相似的 token 排在一起，沿排序序列做 pooling 更能保持内容相似 token 的邻接关系。

中频分支：

```text
每 2 个相邻 token 聚合为 1 个 token
```

低频分支：

```text
每 4 个相邻 token 聚合为 1 个 token
```

pooling 的方案是：

```text
基于 x_scores 的 weighted average segment pooling
```

原因是 `x_scores` 表示 token 对所属动态语义原型的归属置信度。高置信 token 通常更接近当前 subgroup 的语义中心，低置信 token 更可能是边界点、混合纹理或噪声点。因此，在中低频聚合时让高置信 token 占更大权重，可以让 pooled token 更稳定地代表该 segment 的主体结构。

对一个长度为 `k` 的 segment：

```text
G_j = {X_{jk}, X_{jk+1}, ..., X_{jk+k-1}}
```

先取对应的置信度：

```text
s_i = x_scores_i
```

再在 segment 内归一化：

```text
w_i = s_i / (sum_{t in G_j} s_t + eps)
```

最后做加权平均：

```text
X_pool[j] = sum_{i in G_j} w_i · X_i
```

对应到中频和低频：

```text
X_mf[j] = weighted_sum(X_sorted[2j], X_sorted[2j+1])
X_lf[j] = weighted_sum(X_sorted[4j], ..., X_sorted[4j+3])
```

其中：

```text
中频分支 k = 2
低频分支 k = 4
```

### 4.4 多尺度标签对齐

由于 `PFSA` 需要使用语义标签构造 mask，中频和低频分支在 pooling 特征的同时，也要同步 pooling 标签。

推荐方式：

```text
高频标签：直接使用 sorted_belong_idx
中频标签：每 2 个 token 做众数池化
低频标签：每 4 个 token 做众数池化
```

例如：

```text
[2, 2]       → 2
[2, 3]       → 置信度更高者或第一个主标签
[1, 1, 1, 4] → 1
```

如果一个 pooling segment 内存在多个标签，可以用以下规则打破平局：

1. 选择数量最多的标签。
2. 若数量相同，选择 `x_scores` 总和更高的标签。
3. 若仍相同，选择第一个 token 的标签。

这样可以保证中低频 token 仍然有明确的语义归属。

### 4.5 低到高逐级注入

`LMR` 的关键不是三路并行，而是低到高递进。

第一步，低频分支先完成结构级建模：

```text
Y_lf = PFSA(X_lf)
```

第二步，将低频输出上采样到中频粒度，并注入中频分支：

```text
X_mf_hat = X_mf + gamma_1 * Up(Y_lf)
Y_mf = PFSA(X_mf_hat)
```

第三步，将中频输出上采样到高频粒度，并注入高频分支：

```text
X_hf_hat = X_hf + gamma_2 * Up(Y_mf)
Y_hf = PFSA(X_hf_hat)
```

其中：

```text
gamma_1 # 低频到中频的可学习门控
gamma_2 # 中频到高频的可学习门控
Up      # token 粒度对齐操作
```

`gamma_1` 和 `gamma_2` 初始化为较小值，例如 `0` 或 `0.1`。这样训练初期模型不会被跨尺度注入扰乱，后续再逐渐学习如何利用低频和中频信息。

### 4.6 Token 粒度上采样

这里的 `Up()` 不是像素超分辨率中的图像上采样，而是 token 序列粒度对齐。

例如：

```text
Y_lf: N/4 → N/2
Y_mf: N/2 → N
```

这里使用：

```text
segment-aware repeat unpooling
```

也就是严格按照前面 `Segment Pooling` 的分组关系，把一个 pooled token 复制回它原本对应的更细粒度 segment。这样不需要额外插值假设，也不会改变 `DPR` 排序后的语义邻接关系。

对应到两级上采样：

```text
Y_lf_up = repeat_each(Y_lf, 2) # N/4 → N/2
Y_mf_up = repeat_each(Y_mf, 2) # N/2 → N
```

更具体地：

```text
Y_lf_up[2j]     = Y_lf[j]
Y_lf_up[2j + 1] = Y_lf[j]

Y_mf_up[2j]     = Y_mf[j]
Y_mf_up[2j + 1] = Y_mf[j]
```

这里的 `repeat_each` 应该与 pooling 时的 segment 边界完全一致：

```text
中频 pooling: 2 个 token → 1 个 token
低频 pooling: 4 个 token → 1 个 token

低频到中频上采样: 1 个低频 token → 2 个中频 token
中频到高频上采样: 1 个中频 token → 2 个高频 token
```

注意这里不是把低频 `N/4` 一步直接上采样到高频 `N`，而是遵循 LMR 的递进链路：

```text
Y_lf: N/4
→ repeat_each(..., 2)
→ Y_lf_up: N/2
→ 注入中频分支

Y_mf: N/2
→ repeat_each(..., 2)
→ Y_mf_up: N
→ 注入高频分支
```

如果需要更强表达能力，可以在 repeat 前后加入轻量线性投影：

```text
Y_lf_up = repeat_each(Linear_lf2mf(Y_lf), 2)
Y_mf_up = repeat_each(Linear_mf2hf(Y_mf), 2)
```

如果存在 padding token，需要在上采样后去除多余部分，保证最终长度与目标分支一致。

### 4.7 多级输出融合

完成三路建模后，需要将所有输出恢复到统一 token 粒度：

```text
Y_lf_up: B x N x C_lf
Y_mf_up: B x N x C_mf
Y_hf:    B x N x C_hf
```

然后拼接并投影：

```text
Y = Proj(Concat(Y_lf_up, Y_mf_up, Y_hf))
```

最后利用 `DPR` 输出的 `idx_last` 恢复原始 token 顺序：

```text
Y_original = gather(Y, idx_last)
```

再 reshape 回图像特征形式：

```text
B x N x C → B x C x H x W
```

之后进入 Block 内的投影、残差和前馈层。

### 4.8 LMR 的工程注意事项

`LMR` 实现时需要重点注意：

1. 三个分支的 token 长度不同，attention state 必须分别维护。
2. pooling 特征时必须同步 pooling 标签和 `x_scores`。
3. 低频到中频、中频到高频的上采样长度必须严格对齐。
4. `gamma_1` 和 `gamma_2` 使用可学习参数，并小值初始化。
5. 多级输出融合前，三路特征必须恢复到相同 token 数。
6. 融合后必须用 `idx_last` 恢复原始空间顺序。
7. 若使用 padding，需要保证 padding token 不参与有效注意力或最终输出。

## 5. 三个模块的接口衔接

三大模块之间的核心接口如下：

```text
DPR 输出：
idx_last
sorted_belong_idx
x_scores
prototypes
```

这些信息被后续模块使用：

```text
idx_last           → LMR 输出后恢复原始空间顺序
sorted_belong_idx → PFSA 构造 cluster-aware mask
x_scores          → PFSA 计算 purity / var / 动态焦点率
prototypes        → 全局中心交互分支生成 k_global / v_global
```

`LMR` 在低频、中频、高频三个尺度上分别调用 `PFSA`：

```text
Y_lf = PFSA(X_lf, labels_lf, scores_lf, attn_state_lf)
Y_mf = PFSA(X_mf_hat, labels_mf, scores_mf, attn_state_mf)
Y_hf = PFSA(X_hf_hat, labels_hf, scores_hf, attn_state_hf)
```

然后返回当前 Block 的三级注意力状态：

```python
current_attn_state = {
    "lf": attn_lf,
    "mf": attn_mf,
    "hf": attn_hf,
}
```

供下一个 Block 继续使用。

## 6. 推荐前向伪代码

单个 `CAPANet Block` 可以抽象为：

```python
def forward(self, x, prev_attn_state=None):
    x_norm = self.norm(x)

    idx_last, sorted_belong_idx, x_scores, prototypes = self.dpr(x_norm)

    k_global, v_global = self.global_branch(
        x_norm,
        prototypes,
    )

    y, current_attn_state = self.lmr_pfsa(
        x_norm,
        idx_last=idx_last,
        sorted_belong_idx=sorted_belong_idx,
        x_scores=x_scores,
        k_global=k_global,
        v_global=v_global,
        prev_attn_state=prev_attn_state,
    )

    y = self.proj(y)
    x = x + y
    x = x + self.ffn(x)

    return x, current_attn_state
```

网络级流程可以写为：

```python
attn_state = {
    "lf": None,
    "mf": None,
    "hf": None,
}

for block in self.blocks:
    x, attn_state = block(x, attn_state)

x = self.reconstruction_head(x)
```

## 7. 总结

`DPR`、`PFSA` 和 `LMR` 的关系可以概括为：

```text
DPR：内容组织
PFSA：连接筛选
LMR：层次重建
```

更具体地说：

1. `DPR` 提供内容感知 token 组织能力，将空间分散但语义相近的 token 聚合为合理候选集合。
2. `PFSA` 在候选集合内进行渐进聚焦，只保留对重建最有价值的关键连接。
3. `LMR` 将有效交互放入低频、中频、高频三个层级，形成由结构到纹理的递进式重建路径。

因此，该方案的实现重点不是单独加入某一个算子，而是打通如下完整链路：

```text
DPR 输出路由先验
→ PFSA 根据路由先验筛选关键连接
→ LMR 在多级尺度中组织 PFSA
→ Block 间传递三级注意力状态
→ 最终提升轻量级超分辨率的结构连续性与纹理恢复能力
```
