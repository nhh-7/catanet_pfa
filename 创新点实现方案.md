# CAPANet 创新点实现方案

本文从工程落地视角说明 `CAPANet` 的实现路径。方案以现有轻量级图像超分辨率代码框架中的**浅层特征提取、内容组织、全局交互与重建头**为工程基础，将创新点拆分为三个可实现模块：

1. `DPR`：Dynamic Prototype Router，动态语义原型路由
2. `PFSA`：Progressive Focused Sparse Attention，渐进聚焦稀疏注意力
3. `LMR`：Low-to-High Multi-Level Reconstruction，低到高多级协同重建

整体目标是在轻量级预算下完成三件事：

1. 建立跨区域相似 Token 的内容组织能力
2. 建立按重要性筛选连接的稀疏交互能力
3. 建立从结构到纹理的多级递进重建能力

---

## 1. 实现目标与模块命名

建议在工程中采用以下统一命名：

- 网络名称：`CAPANet`
- 基本单元：`CAPANet Block`
- 路由模块：`DPR`
- 稀疏注意力模块：`PFSA`
- 多级重建模块：`LMR`
- 多级注意力状态：`attn_state = {"lf", "mf", "hf"}`

建议的默认超参数如下：

- `level_scales = [1, 2, 4]`
- `level_head_split = [2, 1, 1]`
- `gamma_1, gamma_2`：跨尺度注入门控
- `r`：动态焦点率
- `beta`：注意力先验融合系数

其中：

- `r` 决定每一级保留多少关键连接
- `beta` 决定前序聚焦先验与当前层内容分布的融合比例

---

## 2. 网络级实现入口

在现有轻量级超分辨率主干的基础上，建议继续采用以下外层流程：

1. `shallow feature extraction`
2. 多个 `CAPANet Block`
3. `reconstruction head`

其中，`CAPANet Block` 负责实现 `CAPANet` 的主要创新点。  
网络级推荐采用跨 Block 传递三级注意力状态的方式，让聚焦信息在深层逐步稳定：

```python
attn_state = {"lf": None, "mf": None, "hf": None}

for block in self.blocks:
    x, attn_state = block(x, attn_state)

x = self.reconstruction_head(x)
```

这一组织方式可以让低频、中频、高频三个尺度的注意力先验分别持续演化，而不是在每个 Block 内彼此独立。

---

## 3. CAPANet Block 的接口设计

### 3.1 输入输出形式

推荐接口如下：

```python
def forward(self, x, prev_attn_state=None):
    ...
    return x, current_attn_state
```

其中：

- `x` 为当前 Block 输入特征
- `prev_attn_state` 为来自上一 Block 的多级注意力状态
- `current_attn_state` 为当前 Block 输出的多级注意力状态

### 3.2 Block 内部推荐流程

建议 `CAPANet Block` 内部按照以下顺序组织：

1. `LayerNorm`
2. `DPR` 动态语义路由
3. 全局中心交互分支
4. `LMR + PFSA` 多级聚焦重建
5. `Proj / Conv1x1`
6. `FFN / ConvFFN`
7. 残差连接

对应伪代码如下：

```python
def forward(self, x, prev_attn_state=None):
    x_norm = self.norm(x)

    idx_last, sorted_belong_idx, x_scores, prototypes = self.dpr(x_norm)
    k_global, v_global = self.global_branch(x_norm, prototypes)

    y, current_attn_state = self.lmr_pfsa(
        x_norm,
        idx_last=idx_last,
        sorted_belong_idx=sorted_belong_idx,
        x_scores=x_scores,
        k_global=k_global,
        v_global=v_global,
        prev_attn_state=prev_attn_state,
    )

    y = self.proj(y)
    x = x + y
    x = x + self.ffn(x)
    return x, current_attn_state
```

## 4. DPR：动态语义原型路由实现

### 4.1 输出内容

`DPR` 推荐输出以下四类信息：

- `idx_last`
- `sorted_belong_idx`
- `x_scores`
- `prototypes`

其中：

- `idx_last` 用于恢复原始 Token 顺序
- `sorted_belong_idx` 用于构造 cluster-aware mask
- `x_scores` 用于评估组内纯度与动态焦点率
- `prototypes` 用于后续全局中心交互

### 4.2 实现逻辑

给定输入特征：

$$
X \in \mathbb{R}^{B \times N \times C}
$$

先生成动态原型：

$$
P \in \mathbb{R}^{B \times M \times C}
$$

再计算 Token 与原型之间的关联分数：

$$
S = \text{Softmax}(\phi(X)\psi(P)^T)
$$

随后根据关联分数完成排序、聚合与路由组织。

### 4.3 工程建议

在实现上，`DPR` 可以直接承接现有内容聚类与排序相关的数据结构，只需补齐两点：

1. 将原型张量显式输出，便于后续全局分支共享
2. 将 `x_scores` 保留到注意力模块，供动态焦点率计算使用

---

## 5. PFSA：渐进聚焦稀疏注意力实现

### 5.1 多级注意力状态

推荐在工程中统一维护如下状态结构：

```python
prev_attn_state = {
    "lf": prev_attn_lf,
    "mf": prev_attn_mf,
    "hf": prev_attn_hf,
}
```

每个尺度都维护自己的注意力先验，便于逐层聚焦。

### 5.2 动态焦点率

对每一级、每个 subgroup 计算：

- `purity_l`：当前 subgroup 中同类标签的占比
- `var_l`：当前 subgroup 的 `x_scores` 方差

动态焦点率建议定义为：

```text
r_l = clip(r_base + lambda_p * purity_l - lambda_v * var_l, r_min, r_max)
```

推荐默认值：

- `r_base = 0.5`
- `r_min = 0.25`
- `r_max = 0.75`

该设计使模型具备以下性质：

1. 组内越纯，保留的关键连接越多
2. 组内越混杂，交互范围越窄
3. 聚焦范围随输入内容动态变化

### 5.3 注意力先验融合

每一级的注意力分布建议使用如下方式更新：

```text
A_l = beta * A_{l-1} + (1 - beta) * A_l^{cur}
```

随后根据 `A_l` 与 `r_l` 共同生成稀疏连接掩码：

```text
M_l = TopK(A_l, r_l)
```

最终执行带掩码的稀疏注意力：

```text
Y_l = Attention(Q_l, K_l, V_l; M_l)
```

### 5.4 工程建议

若当前工程已具备稀疏矩阵乘法或其他高效掩码注意力实现，可直接作为 `PFSA` 的计算后端。  
实现时建议优先保证：

1. 稀疏掩码支持 batch 内并行
2. 掩码生成和注意力归一化过程数值稳定
3. 多级状态的缓存与回传不会引入额外张量错位

## 6. LMR：低到高多级协同重建实现

### 6.1 多级分支拆分

排序后的 Token 先在通道维上拆分为三组：

- 高频级 `X_hf`：2 个 head，保持原始 Token 粒度
- 中频级 `X_mf`：1 个 head，执行 `2x` token pooling
- 低频级 `X_lf`：1 个 head，执行 `4x` token pooling

这里建议使用 `segment pooling`，即在排序后的 subgroup 内对相邻 Token 做局部聚合，以保持内容相似 Token 的邻接关系。

### 6.2 多尺度标签对齐

为了让中频级和低频级继续使用内容标签构造掩码，建议同步池化标签：

- 高频级：直接使用 `sorted_belong_idx`
- 中频级：每 `2` 个 Token 做众数池化
- 低频级：每 `4` 个 Token 做众数池化

### 6.3 低到高逐级注入

多级数据流严格按照低频到高频顺序执行：

1. 低频级先完成结构级聚焦建模
2. 将低频输出上采样后注入中频级
3. 中频级完成过渡级聚焦建模
4. 将中频输出上采样后注入高频级
5. 高频级完成细节级聚焦建模

核心公式如下：

```text
X_mf_hat = X_mf + gamma_1 * Up(Y_lf)
X_hf_hat = X_hf + gamma_2 * Up(Y_mf)
```

其中 `gamma_1` 和 `gamma_2` 为可学习门控参数。

### 6.4 多级输出融合

三路输出恢复到统一粒度后进行融合：

$$
Y = \text{Proj}(\text{Concat}(Y^{lf}, Y^{mf}, Y^{hf}))
$$

融合后输出到后续投影层与前馈层。

### 6.5 全局分支接入

若工程中已有全局中心交互模块，建议按以下方式接入：

- 高频级：接入全局分支
- 中频级：接入全局分支
- 低频级：不接入全局分支

这样可以让高分辨率分支获得额外全局先验，同时避免低频分支出现过度冗余的全局信息叠加。

## 7. 单个 CAPANet Block 的完整前向流程

推荐实现顺序如下：

1. `LayerNorm`
2. `DPR` 生成原型、排序索引、标签与得分
3. 全局中心交互分支生成 `k_global, v_global`
4. `LMR + PFSA` 完成多级聚焦交互
5. `Proj / Conv1x1`
6. `FFN / ConvFFN`
7. 残差输出

伪代码如下：

```python
def forward(self, x, prev_attn_state=None):
```

## 8. 复杂度控制目标

`CAPANet` 的工程实现应始终围绕轻量级目标展开。建议控制在以下区间内：

- 参数量：约 `0.55M - 0.60M`
- FLOPs：约 `46G` 左右
- 延迟：单张推理保持在轻量级部署可接受范围内

实现时可通过以下方式控制复杂度：

1. 保持总 head 数不变
2. 让中频与低频分支在更短序列上完成注意力计算
3. 将新增参数集中在门控、投影与少量融合层
4. 优先使用稀疏注意力后端减少无效计算

## 9. 分阶段实现建议

建议按四个阶段推进：

### 9.1 第一阶段：DPR 路由跑通

目标：

1. 输出稳定的 `idx_last`
2. 输出正确的 `sorted_belong_idx`
3. 输出可用于后续调度的 `x_scores` 与 `prototypes`

### 9.2 第二阶段：PFSA 跑通

目标：

1. 实现多级注意力状态缓存
2. 实现动态焦点率计算
3. 实现稀疏掩码生成与注意力计算

### 9.3 第三阶段：LMR 跑通

目标：

1. 实现三路分支拆分
2. 实现多尺度 pooling 与标签对齐
3. 实现低到高逐级注入
4. 实现统一融合输出

### 9.4 第四阶段：训练与验证

目标：

1. 完成端到端训练
2. 跑通 `PSNR / SSIM / Params / FLOPs / Latency`
3. 补齐 `w/o DPR`、`w/o PFSA`、`w/o LMR` 等消融实验
4. 输出注意力图与多级特征可视化

---

## 10. 实现方案总结

`CAPANet` 的实现重点不是单一算子的引入，而是将以下三类能力在同一工程框架内协同落地：

1. `DPR` 提供内容感知的 Token 组织能力
2. `PFSA` 提供渐进聚焦的关键连接筛选能力
3. `LMR` 提供从结构到纹理的低到高递进重建能力

因此，这份实现方案的核心目标可以概括为：

> 在现有轻量级图像超分辨率工程框架的基础上，完成 `CAPANet` 在内容路由、稀疏聚焦与多级重建三项关键机制上的统一实现。
