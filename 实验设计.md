# CAPANet 实验设计方案

本文用于规划 `CAPANet` 论文中的实验部分。实验设计围绕课题主张展开：在轻量级图像超分辨率中，仅提高模型容量并不是关键，更重要的是在有限计算预算下完成有效的内容组织、关键连接筛选和层次化重建。  

因此，实验需要证明的不只是 `CAPANet` 的指标更高，还要证明三个核心机制的必要性：

1. `DPR`（Dynamic Prototype Router）能够把空间上分散但内容相似的 token 组织到更有效的候选集合中。
2. `PFSA`（Progressive Focused Sparse Attention）能够在候选集合内进一步筛选关键连接，减少无效注意力交互。
3. `LMR`（Low-to-High Multi-Level Reconstruction）能够按照低频结构、中频过渡和高频纹理的顺序完成更稳定的递进式重建。

实验整体采用“主结果证明性能优势，消融证明模块贡献，分析实验解释机制有效性，复杂度实验证明轻量部署价值”的结构。

---

## 1. 实验目标与证据链

### 1.1 核心问题

CAPANet 要回答以下三个问题：

1. 在轻量级参数规模下，CAPANet 是否能优于主流 CNN、Transformer 和内容聚合类超分辨率方法。
2. DPR、PFSA、LMR 是否分别解决了内容组织、连接筛选和层次重建问题，而不是简单增加参数带来的收益。
3. CAPANet 的优势是否主要体现在复杂结构和高频纹理场景中，例如 `Urban100` 的建筑线条、重复窗格，以及 `Manga109` 的漫画线稿和纹理区域。

### 1.2 推荐实验逻辑

论文实验部分建议按以下顺序组织：

1. **定量主对比**：证明 CAPANet 在 `x2 / x3 / x4` 上具备整体性能优势。
2. **复杂度对比**：证明性能提升没有以明显更高参数量和时延为代价。
3. **核心模块消融**：证明 DPR、PFSA、LMR 各自有效，并且联合使用收益最大。
4. **关键设计消融**：验证动态焦点率、head 分配、注意力继承和全局分支位置的合理性。
5. **可解释性分析**：展示路由分组、稀疏注意力和低到高重建过程确实符合方法动机。
6. **定性可视化**：用复杂样例说明 CAPANet 对结构连续性和纹理真实性的改善。

## 2. 统一实验设置

### 2.1 数据集

主实验建议以 `DIV2K` 作为训练集，原因是当前代码配置、已有结果表和 CATANet/PFT 等轻量级方法的公开对比设置更容易保持一致。

| 类型 | 数据集 | 用途 |
| :--- | :--- | :--- |
| 训练集 | `DIV2K` | 主对比实验、消融实验、复杂度对比的统一训练来源 |
| 可选扩展训练集 | `DF2K = DIV2K + Flickr2K` | 作为补充实验，用于验证更大训练集下的泛化上限，不作为主表默认设置 |
| 测试集 | `Set5`、`Set14`、`B100`、`Urban100`、`Manga109` | 标准 SISR 测试基准 |

为了避免与已有轻量级方法产生不公平比较，主文中应明确标注训练集。若主结果采用 `DIV2K`，则不要在同一主表中混入 `DF2K` 训练结果；若补充 `DF2K`，建议单独成表。

### 2.2 放大倍率

主实验覆盖：

- `x2`
- `x3`
- `x4`

其中 `x4` 作为消融和分析实验的默认倍率。原因是 `x4` 退化更强，更容易暴露结构断裂、纹理伪影和注意力冗余问题，也更适合检验 DPR、PFSA、LMR 的贡献。

### 2.3 评价指标

主指标：

- `PSNR`
- `SSIM`

统一在 `YCbCr` 的 `Y` 通道上计算，并根据放大倍率裁剪边界：

| 倍率 | crop border |
| :---: | :---: |
| `x2` | 2 |
| `x3` | 3 |
| `x4` | 4 |

复杂度指标：

- `Params`
- `Multi-Adds / FLOPs`
- `Latency`

复杂度统一使用输入尺寸 `3 x 256 x 256` 统计。由于 CAPANet 包含 `matmul / einsum` 等注意力计算，论文中建议说明 FLOPs 统计口径：主表采用包含 `Conv2d / Linear / matmul / einsum` 的完整前向统计；若引用仅统计 `Conv2d + Linear` 的结果，应放入注释而不是主表。

### 2.4 训练配置

与当前工程配置保持一致，建议写为：

| 倍率 | HR patch | 等价 LR patch | batch size | 训练策略 | 迭代数 |
| :---: | :---: | :---: | :---: | :--- | :---: |
| `x2` | `128 x 128` | `64 x 64` | `4 x 16` | scratch | `800K` |
| `x3` | `192 x 192` | `64 x 64` | `4 x 16` | finetune | `250K` |
| `x4` | `256 x 256` | `64 x 64` | `4 x 16` | finetune | `250K` |

基础训练细节：

- 优化器：`Adam`
- 初始学习率：`2e-4`
- `betas = (0.9, 0.99)`
- 损失函数：`L1 loss`
- 数据增强：随机水平翻转、随机旋转
- 随机种子：`3407`
- 学习率调度：`MultiStepLR`

消融实验统一使用 `x4 / DIV2K / 250K`。如果为了节省成本采用更短训练步数，需要在表注中明确，并保证所有消融模型训练步数完全一致。

### 2.5 默认模型配置

CAPANet 默认配置如下：

| 配置项 | 默认值 | 作用 |
| :--- | :--- | :--- |
| `routing_mode` | `dpr` | 使用动态语义原型路由 |
| `focus_mode` | `dynamic` | 使用内容感知动态焦点率 |
| `use_sparse_pfsa` | `true` | 启用 PFSA 稀疏连接筛选 |
| `level_head_split` | `[2, 1, 1]` | 高频、中频、低频 head 分配 |
| `global_branch_levels` | `[mf, hf]` | 全局原型分支接入中频和高频 |
| `attn_state_mode` | `layered` | 低频、中频、高频分别继承注意力状态 |

其中 `level_head_split = [2, 1, 1]` 的顺序为 `(HF, MF, LF)`，与代码中的 `LEVEL_NAMES` 保持一致。

## 3. 主实验：与轻量级 SOTA 对比

### 3.1 对比方法选择

主表建议覆盖三类方法：

| 类型 | 方法 |
| :--- | :--- |
| 轻量 CNN | `CARN`、`IMDN`、`RFDN`、`AWSRAN-M`、`OSFFNet` |
| 轻量 Transformer / 混合模型 | `ESRT`、`ELAN-light`、`SwinIR-light`、`SPIN` |
| 内容组织与聚焦注意力相关方法 | `CATANet`、`PFT-light` |

若版面有限，主文可保留与已有结果表一致的轻量方法集合，并将 `CATANet`、`PFT-light` 作为重点比较对象。原因是它们分别对应 CAPANet 的两个直接上下文：内容感知 token 聚合与渐进聚焦注意力。

### 3.2 定量主表设计

建议分别建立 `x2 / x3 / x4` 三张表，每张表包含：

- `Params`
- `Set5 PSNR/SSIM`
- `Set14 PSNR/SSIM`
- `B100 PSNR/SSIM`
- `Urban100 PSNR/SSIM`
- `Manga109 PSNR/SSIM`

表格解读重点不应只说“平均更高”，而应按数据集性质解释：

1. `Set5`、`Set14`、`B100` 用于证明基础恢复能力稳定。
2. `Urban100` 用于证明规则结构、建筑线条和长程重复纹理恢复更好。
3. `Manga109` 用于证明细线条、漫画纹理和高频边界恢复更强。

结合当前已有结果，CAPANet 的主要叙述重点应放在 `Urban100` 和 `Manga109`。例如 `x4` 下，CAPANet 相比同表轻量方法在复杂结构数据集上优势更明显，能够支撑“内容组织 + 聚焦连接 + 多级重建”对复杂纹理更有效的结论。

## 4. 复杂度与部署友好性实验

### 4.1 复杂度主表

复杂度表建议放在主对比之后，重点比较：

- `SwinIR-light`
- `CATANet-L`
- `PFT-light`
- `CAPANet w/o LMR`
- `CAPANet`

建议表头：

| Model | Params | Multi-Adds | Latency |
| :--- | :---: | :---: | :---: |

当前已有结果显示，CAPANet 参数量约 `617K`，完整 FLOPs 约 `59.8G`，Latency 约 `85.1ms`。这组结果的论文解释应强调：

1. CAPANet 的参数规模仍处于轻量级范围。
2. 相比 PFT-light，CAPANet 在 FLOPs 更低或接近的情况下获得更好的复杂结构恢复表现。
3. 相比 CATANet-L，CAPANet 引入 LMR 后 FLOPs 增加，但时延基本保持接近，说明新增计算对实际推理延迟的影响有限。

### 4.2 时延测试规范

为了让复杂度实验更可信，建议统一说明：

- GPU：单张 `RTX 4090`
- 输入尺寸：`3 x 256 x 256`
- batch size：`1`
- warm-up：至少 `50` 次
- 正式统计：至少 `100` 次前向平均
- 使用 `torch.cuda.synchronize()` 保证计时准确

若论文中没有实际重复计时脚本，应避免过度强调精确时延，只把 Latency 作为辅助指标。

## 5. 核心模块消融实验

消融实验统一采用 `x4 / DIV2K / 相同训练步数`，重点报告 `Set5`、`Urban100`、`Manga109`。其中 `Set5` 表示基础重建稳定性，`Urban100` 和 `Manga109` 表示复杂结构与高频纹理恢复能力。

### 5.1 DPR、PFSA、LMR 主消融

建议使用以下五组模型：

| Model | Components | 目的 |
| :---: | :--- | :--- |
| A | `Backbone + Global Branch + Reconstruction Head` | 基础模型 |
| B | `A + DPR` | 验证动态语义原型路由 |
| C | `B + PFSA` | 验证稀疏聚焦连接筛选 |
| D | `B + LMR` | 验证低到高多级重建 |
| E | `B + PFSA + LMR` | 完整 CAPANet |

这一组是论文最重要的消融。解读时建议按三个层次展开：

1. `A -> B`：如果 `Urban100 / Manga109` 提升明显，说明 DPR 对跨区域相似纹理组织有效。
2. `B -> C`：如果 FLOPs 不增加或降低且指标提升，说明 PFSA 的收益来自连接筛选，而不是单纯增大模型。
3. `B -> D` 与 `C/D -> E`：如果完整模型最好，说明 LMR 与 PFSA 不是互斥设计，而是一个负责筛选连接，一个负责组织重建层次。

### 5.2 对照设计要求

核心模块消融必须保持以下条件一致：

- 相同训练集
- 相同训练迭代数
- 相同输入 patch
- 相同优化器和学习率调度
- 相同测试脚本

如果某个消融会改变参数量和 FLOPs，应在表中同时列出 `Params` 和 `FLOPs`，避免被误解为性能提升完全来自模型变大。

## 6. 关键设计消融实验

### 6.1 动态焦点率消融

目的：证明动态 `r` 的价值不是简单选择一个固定稀疏比例，而是根据内容纯度自适应分配注意力连接。

推荐分组：

| Focus Strategy | 说明 |
| :--- | :--- |
| `Fixed r = 0.25` | 高稀疏，可能丢失有效连接 |
| `Fixed r = 0.50` | 中等稀疏 |
| `Fixed r = 0.75` | 低稀疏，可能引入冗余连接 |
| `Dynamic r` | 根据 purity 和 score variance 自适应调整 |

建议报告：

- `Avg. Keep Ratio`
- `FLOPs`
- `Urban100 PSNR`
- `Manga109 PSNR`

预期结论：

1. 固定过低的 `r` 会损失细节恢复能力。
2. 固定过高的 `r` 会重新引入无效连接和注意力噪声。
3. `Dynamic r` 在平均保留比例不最高的情况下取得更好结果，说明它改善的是连接质量而不是连接数量。

### 6.2 Head 分配策略消融

目的：证明 `HF / MF / LF = [2, 1, 1]` 是结构与纹理之间更合理的预算分配。

推荐分组：

| Head Split `(HF, MF, LF)` | 含义 |
| :--- | :--- |
| `[4, 0, 0]` | 全部用于高频，缺少结构级建模 |
| `[3, 1, 0]` | 高频为主，缺少低频结构分支 |
| `[2, 1, 1]` | 默认配置，兼顾纹理、边界与结构 |
| `[1, 1, 2]` | 偏重低频，可能削弱细节恢复 |

预期结论：

1. 只建模高频容易造成结构连续性不足。
2. 过度偏向低频会削弱细线条和纹理恢复。
3. `[2, 1, 1]` 能在 Urban100 和 Manga109 上取得更好平衡。

### 6.3 注意力继承策略消融

目的：证明低频、中频、高频分别维护注意力状态比共享单一状态更合理。

推荐分组：

| Strategy | 说明 |
| :--- | :--- |
| `No Inheritance` | 每个 Block 从零计算注意力 |
| `Shared Single-State` | 所有尺度共享同一个前序注意力先验 |
| `Level-wise prev_attn_state` | 每个尺度分别继承自己的注意力状态 |

预期结论：

1. 无继承会使注意力每层重新搜索，聚焦稳定性不足。
2. 单状态继承会混淆不同尺度的结构与纹理先验。
3. 分层继承更符合 LMR 的低频、中频、高频分工。

### 6.4 全局原型分支位置消融

目的：验证全局中心弱分支接入中频和高频最合理。

推荐分组：

| Global Branch Placement | 说明 |
| :--- | :--- |
| `High only` | 只辅助高频纹理恢复 |
| `High + Mid` | 默认配置 |
| `High + Mid + Low` | 低频也接入全局分支 |

预期结论：

1. 高频单独接入不足以修正结构边界和区域过渡。
2. 高频 + 中频接入能同时服务边界过渡和细节补偿。
3. 低频继续接入全局分支可能带来冗余，因为低频分支本身更偏结构级聚合。

### 6.5 路由方式消融

建议补充一组路由方式消融，用于证明 DPR 相比普通内容排序或固定分组更有必要：

| Routing Mode | 说明 |
| :--- | :--- |
| `Window / Regular Grouping` | 固定空间分组 |
| `Content Sorting without Prototype` | 只按内容相似排序，不显式生成原型 |
| `DPR` | 动态语义原型 + 归属置信度 + 路由元信息 |

这一组实验可以作为补充材料。如果篇幅有限，主文保留 DPR 主消融即可。

## 7. 解释性与机制分析实验

解释性实验不一定需要大规模训练，但需要服务于论文机制说明。建议重点做以下三类。

### 7.1 DPR 路由质量分析

目的：证明 DPR 确实把相似内容组织到一起，而不是产生随机分组。

建议统计：

- `cluster purity`
- 每个 prototype 对应 token 数量分布
- `x_scores` 均值与方差
- 同一 prototype 内 token 的空间分布可视化

推荐样例：

- `Urban100`: 建筑窗格、栏杆、重复线条
- `Manga109`: 漫画边缘线、网点纹理、文字附近区域

预期现象：

1. 同一 prototype 会聚集相似纹理或结构 token。
2. 边界混杂区域的 `x_scores` 方差更大。
3. DPR 可以建立远距离重复区域之间的候选关系。

### 7.2 动态焦点率与内容纯度关系

目的：证明动态 `r` 的调整具有内容依据。

建议绘制散点图或分桶统计：

- 横轴：`cluster purity`
- 横轴补充：`x_scores variance`
- 纵轴：动态 `r`
- 颜色：数据集或图像区域类型

预期结论：

1. 组内内容越纯，动态 `r` 越高，允许更多有效连接。
2. 组内置信度方差越大，动态 `r` 越低，减少不稳定连接传播。
3. 动态焦点率形成“纯组更密、混组更稀”的分配规律。

### 7.3 多级注意力可视化

目的：证明 LMR 的低频、中频、高频分支具有不同功能。

建议对同一 query token 可视化三类注意力：

| 分支 | 期望关注区域 |
| :--- | :--- |
| `LF` | 大范围结构、主体轮廓、重复布局 |
| `MF` | 区域边界、线条过渡、中尺度纹理 |
| `HF` | 细边缘、纹理细节、密集线稿 |

可视化方式：

- attention heatmap
- top-k 连接位置图
- 不同尺度分支的响应叠加图

预期结论：

LMR 不是简单并行多分支，而是形成由低频结构到高频细节的递进重建链路。

## 8. 定性可视化实验

### 8.1 样例选择

建议重点选择复杂结构样例：

| 数据集 | 推荐样例类型 |
| :--- | :--- |
| `Urban100` | 建筑窗格、栅栏、斜线结构、重复纹理 |
| `Manga109` | 人物轮廓、漫画线稿、网点纹理、文字边缘 |
| `Set14 / B100` | 自然图像边缘和中等纹理区域，作为稳定性补充 |

如果使用具体文件名，可以优先考虑：

- `Urban100`: `img_004`、`img_076`、`img_092`
- `Manga109`: `YumeiroCooking`、`AisazuNihaIrarenai`、`BurstAngel`

### 8.2 可视化内容

每个样例建议展示：

1. LR 或 Bicubic 输入
2. 对比方法结果，例如 `CATANet`、`PFT-light`
3. `CAPANet` 结果
4. HR ground truth
5. 局部放大框
6. error map 或残差热力图

定性讨论重点：

1. CAPANet 是否减少建筑线条断裂。
2. CAPANet 是否减少漫画线稿粘连和伪纹理。
3. CAPANet 是否在重复纹理区域保持更好的方向一致性。
4. CAPANet 是否避免过度锐化造成的噪声放大。

## 9. 补充实验建议

### 9.1 DF2K 扩展训练

如果时间和算力允许，可以增加一组 `DF2K` 训练结果，但建议放在补充实验中，而不是替代主实验。

目的：

1. 验证 CAPANet 在更大训练集上的性能上限。
2. 观察 DPR 对更多样纹理分布的适应能力。
3. 避免主表中训练集不一致造成对比争议。

### 9.2 轻量配置与速度配置

可以设计两个额外模型：

| 版本 | 设计目的 |
| :--- | :--- |
| `CAPANet-Tiny` | 更低参数和时延，面向部署 |
| `CAPANet` | 默认主模型 |

可调整项：

- 减少 block 数
- 降低 embedding dim
- 调整 `level_head_split`
- 降低 prototype 数量
- 增大稀疏性，即降低 `r_base` 或 `r_max`

这组实验适合作为应用部署讨论，不是当前论文主证据链的必要实验。

### 9.3 失效案例分析

## 12. 推荐结论表达

最终实验结论建议收束为四句话：

1. CAPANet 在 `x2 / x3 / x4` 标准基准上保持稳定性能优势，尤其在 `Urban100` 和 `Manga109` 上提升更明显。
2. DPR 能够提供更有效的内容候选组织，PFSA 能够筛选更高价值的注意力连接，LMR 能够将结构、边界和纹理恢复组织为低到高的递进过程。
3. 动态焦点率、分层注意力继承和 `[2, 1, 1]` 的 head 分配共同说明 CAPANet 的收益来自内容自适应和层次化设计，而不是简单堆叠计算量。
4. 在轻量级参数规模和可接受推理时延下，CAPANet 为复杂结构场景中的高质量超分辨率重建提供了更合理的性能-效率平衡。
