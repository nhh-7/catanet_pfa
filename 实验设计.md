# CAPA-SR 论文实验设计方案

本文面向 `CAPA-SR` 的论文写作与实验组织，目标是围绕该方法的三个核心机制构建完整证据链：

1. `DPR`：Dynamic Prototype Router，动态语义原型路由
2. `PFSA`：Progressive Focused Sparse Attention，渐进聚焦稀疏注意力
3. `LMR`：Low-to-High Multi-Level Reconstruction，低到高多级协同重建

实验设计的核心任务是证明三点：

1. `CAPA-SR` 能在轻量级预算下提升复杂结构与高频纹理的重建质量。
2. `DPR + PFSA + LMR` 的联合设计优于单一内容组织、单一稀疏交互或单尺度注意力方案。
3. `CAPA-SR` 在 `PSNR / SSIM / Params / FLOPs / Latency` 多个维度上具备稳定的性能-效率优势。

本文统一使用以下符号：

- `r`：Dynamic Focus Ratio，表示每一级实际保留的稀疏连接比例。
- `beta`：Attention Inheritance Weight，表示前序注意力先验与当前层注意力分布的融合权重。
- `gamma_1, gamma_2`：低频到中频、中频到高频的可学习门控系数。
- `level_scales = [1, 2, 4]`，`head_split = [2, 1, 1]`：分别对应高频、中频、低频三条分支。

---

## 1. 论文要证明什么

围绕 `CAPA-SR`，本文建议重点证明以下四项结论：

1. `DPR` 能有效建立跨区域相似 Token 的组织能力，提升远距离重复纹理的交互质量。
2. `PFSA` 能通过动态焦点率与注意力继承机制减少无关连接，使注意力计算更聚焦、更高效。
3. `LMR` 能通过低到高逐级注入增强结构连续性与纹理恢复能力，尤其对 `Urban100` 和 `Manga109` 更有帮助。
4. `CAPA-SR` 在保持轻量级参数规模的同时，在主流轻量级超分辨率模型中取得更优的整体性能-效率平衡。

---

## 2. 实验设置

### 2.1 数据集

- 训练集：`DF2K = DIV2K(800) + Flickr2K(2650)`。
- 测试集：`Set5`、`Set14`、`BSD100`、`Urban100`、`Manga109`。
- 放大倍率：`x2 / x3 / x4`。

### 2.2 评价指标

- 重建指标：`PSNR / SSIM`，统一在 `YCbCr` 的 `Y` 通道上计算。
- 复杂度指标：`Params`、`FLOPs`、`Latency`。
- 延迟测试：统一在单张 `RTX 4090` 上进行，输入尺寸固定为 `3 x 256 x 256`。

### 2.3 训练细节

- LR patch size：`64 x 64`。
- Batch size：`32`。
- 优化器：`AdamW`，`betas=(0.9, 0.999)`，`weight_decay=1e-4`。
- 初始学习率：`5e-4`，采用 cosine decay。
- 总训练步数：主实验 `500K`，消融实验统一 `300K`。
- 数据增强：随机翻转、随机旋转、随机 RGB shuffle。

### 2.4 方法配置

- 多级分支：高频级 2 个 head，中频级 1 个 head，低频级 1 个 head。
- 高频级与中频级接入全局交互分支，低频级仅负责结构级聚焦建模。
- 多尺度标签在中频和低频分支分别采用 `2-token` 与 `4-token` 众数池化。
- 主实验默认 `beta = 0.5`，动态焦点率范围设为 `r in [0.25, 0.75]`。

## 3. 主实验：与轻量级 SOTA 的对比

### 3.1 定量对比

对比对象建议覆盖三类方法：

- 经典轻量 CNN：`CARN`、`IMDN`、`RFDN`、`OSFFNet`。
- 轻量 Transformer：`SwinIR-light`、`ELAN-light`、`ATD-light`、`IPG-Tiny`。
- 内容组织或稀疏交互相关方法：`CATANet`、`PFT-light`。

主结论建议围绕以下几个方面展开：

1. 在 `x2 / x3 / x4` 三个倍率下，`CAPA-SR` 在主要测试集上取得稳定领先或并列领先结果。
2. 在 `Urban100` 和 `Manga109` 上，`CAPA-SR` 对重复结构、几何线条和复杂纹理的恢复能力更突出。
3. 在接近轻量级参数量与 FLOPs 的条件下，`CAPA-SR` 仍能保持优于多数轻量 Transformer 的精度表现。
4. 在 `Set5 / Set14 / BSD100` 这类相对平滑的数据集上，`CAPA-SR` 保持稳定收益而不过度牺牲效率。

### 3.2 定性可视化

重点展示下列样例：

- `Urban100`: `img_004`, `img_092`, `img_076`
- `Manga109`: `YumeiroCooking`, `AisazuNihaIrarenai`, `BurstAngel`

建议输出四类可视化：

- 局部放大重建图
- Error map
- 高频、中频、低频三级注意力热点图
- 动态焦点率 `r` 的空间分布图

预期现象如下：

1. 高频分支主要负责细线条、边缘和密集纹理重建。
2. 低频分支更偏向大结构与长距离重复模式建模。
3. 中频分支起到结构边界过渡和尺度衔接作用。
4. `CAPA-SR` 在复杂结构区域能显著减少断裂、粘连和伪纹理。

## 4. 核心消融实验

所有消融统一采用 `x4 / DF2K / 300K` 设置，避免训练时长差异影响结论。

### 4.1 核心模块消融

建议使用以下五组模型，完整展示三个核心机制的独立贡献：

1. `Model A`: `Backbone + Global Branch + Reconstruction Head`
2. `Model B`: `A + DPR`
3. `Model C`: `B + PFSA`
4. `Model D`: `B + LMR`
5. `Model E`: `B + PFSA + LMR`，即完整 `CAPA-SR`

这一组实验用于回答三个问题：

1. 仅有内容组织是否足够。
2. 稀疏聚焦与多级建模各自带来多大收益。
3. 三者联合后是否存在显著的协同增益。

### 4.2 动态焦点率消融

直接分析动态 `r` 的作用：

- 固定 `r = 0.25`
- 固定 `r = 0.50`
- 固定 `r = 0.75`
- `Dynamic r`（本文默认）

建议同时报告：

- `Urban100 / Manga109` 的 PSNR
- 对应 FLOPs
- 平均保留连接比例

核心目标是证明：动态 `r` 的价值不只是“更稀疏”，而是把计算集中到更重要、更纯净的连接上。

### 4.3 多级 head 分配策略消融

比较不同 head 分配：

- `[4, 0, 0]`：单级高频建模
- `[3, 1, 0]`：双级建模
- `[2, 1, 1]`：本文默认
- `[1, 1, 2]`：偏重低频建模

该实验用于证明 `2-1-1` 在纹理恢复能力与全局结构建模能力之间取得更优平衡。

### 4.4 注意力继承机制消融

比较三种继承方式：

- 不继承
- 共享单状态继承
- 分层状态继承：`prev_attn_state = {lf, mf, hf}`，即本文默认

这一实验用于证明：在多级结构下，按尺度分别维护注意力先验比统一共享一张注意力图更合理。

### 4.5 全局分支接入位置消融

比较以下三种方案：

- 仅高频分支接入全局分支
- 高频 + 中频分支接入全局分支
- 高频 + 中频 + 低频分支全部接入全局分支

预期结论是：高频与中频保留全局交互最优，低频分支继续引入全局分支会带来冗余。

## 5. 模型分析与解释性实验

### 5.1 三级注意力的演化过程

从同一个 query token 出发，同时可视化：

- 低频级注意力：关注长距离结构与全局轮廓
- 中频级注意力：关注结构边界与区域过渡
- 高频级注意力：关注边缘、细线条和高频纹理

该实验用于说明多级重建链路并非简单并行分支，而是具有明确的粗到细语义分工。

### 5.2 动态焦点率与内容纯度的关联

对每个 subgroup 统计：

- `cluster purity`
- `x_scores` 方差
- 对应动态 `r`

预期应呈现：

1. purity 越高，`r` 越高
2. 方差越大，`r` 越低
3. 最终形成“纯组更密、混组更稀”的自适应分配模式

### 5.3 性能-复杂度帕累托分析

建议绘制两张图：

- `PSNR vs Params`
- `PSNR vs FLOPs`

需要放入同一坐标系的模型包括：

- `CARN`
- `IMDN`
- `RFDN`
- `CATANet`
- `PFT-light`
- `ATD-light`
- `CAPA-SR`

预期结果是 `CAPA-SR` 位于帕累托前沿左上区域。

## 6. 论文故事线建议

推荐按照以下逻辑组织全文：

1. 动机：轻量级超分辨率仍面临长程依赖不足、注意力冗余严重和单尺度建模受限的问题。
2. 观察：仅靠内容分组无法解决无关连接干扰，仅靠稀疏交互也难以建立从结构到纹理的层次重建链路。
3. 方法：提出 `CAPA-SR`，通过 `DPR`、`PFSA` 和 `LMR` 三个模块统一完成内容组织、关键连接筛选与低到高递进重建。
4. 实验：在多个标准基准上验证方法在重建质量、复杂度和推理效率上的综合优势。
5. 结论：`CAPA-SR` 在保持轻量级预算的同时，显著提升了复杂纹理与规则结构场景下的超分辨率重建能力。
